Sentinel: Anomaly Hunting

Hunt for Process Environment Drift using KQL.

KQL // DEVICEPROCESSEVENTS
DeviceProcessEvents
| where FileName in~ ("lsass.exe", "wininit.exe")
| summarize count() by FileName, DeviceName
| where count() < 5 // Focus on the Long Tail

Tanium: Fleet Memory Sweep

Query Tanium to find the "needle in the haystack" across the fleet.

Hunt Question: "Get Memory Region Protections for [High-Value_Process] from all machines."
  • Identify processes with RWX regions.
  • Verify if regions are Private (not backed by a DLL).

The VAD Tree: Proving Evil

In memory forensics, the VAD tree is the ultimate arbiter of truth.

Mapped Memory

Backed by a file on disk (DLL/EXE). Usually legitimate.

Private Memory

No file backing. Staging ground for Shellcode.

The Hunt & Proof Workflow

  1. Hypothesize: Adversaries are in svchost.exe.
  2. Triage: Use Tanium to sweep for RWX Private regions.
  3. Isolate: Acquire memory from endpoints with "Unbacked" code.
  4. Verify: Use Volatility to carve the MZ header and extract the DLL.

Memory Cheat Sheet

Protection Hunter's Note
READWRITE Hunt for Cleartext Credentials in the heap.
EXECUTE_READ Standard for code. Check for Unbacked modules.
EXECUTE_RW HIGH FIDELITY INDICATOR of injected shellcode.
