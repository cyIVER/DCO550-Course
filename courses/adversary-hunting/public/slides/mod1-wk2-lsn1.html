What is Persistence?

  • Definition: Persistence is any mechanism an adversary uses to maintain access to a system across restarts, changed credentials, and interruptions.
  • Adversary Goal: To survive a reboot. Initial access is fleeting; persistence is long-term control. (The digital key to the house).
  • Hunter's Goal: Persistence artifacts are high-value, relatively static targets. While malware may vanish from memory, the mechanism to launch it again must be stored on disk or OS configuration.

MITRE ATT&CK Tactic: TA0003 - Persistence. This is one of the most populated tactics, with dozens of documented techniques.

The Persistence Landscape

Adversaries want to blend in by using legitimate Windows features for their own ends. We will focus on hunting three of the most abused mechanisms:

1. THE REGISTRY

The central configuration database for the Windows operating system.

Why abused: It is the easiest and most common place to hide an auto-start entry for malicious executables.

2. SERVICES

Background processes that form the backbone of the OS.

Why abused: Adversaries create malicious services or hijack legitimate ones to run with `SYSTEM` level privileges.

3. SCHEDULED TASKS

The native OS job scheduler used for recurring administrative tasks.

Why abused: Perfect for re-launching malware or establishing a regular, timed C2 beacon to bypass heuristic analysis.

*Advanced techniques like WMI persistence and COM hijacking will be covered in later modules.*

Hunting in the Registry

THE TARGETS

The "Run" and "RunOnce" keys are the most common places to look for system (HKLM) and user (HKCU) persistence:

  • HKLM\Software\Microsoft\Windows\CurrentVersion\Run
  • HKLM\Software\Microsoft\Windows\CurrentVersion\RunOnce
  • HKCU\Software\Microsoft\Windows\CurrentVersion\Run
  • HKCU\Software\Microsoft\Windows\CurrentVersion\RunOnce

THE FORENSIC METHOD

We do NOT use regedit. We hunt in offline images.

  • From Memory (.vmem): Use Volatility. windows.registry.hivelist.HiveList finds the virtual address. Then, windows.registry.printkey.PrintKey queries a specific path.
  • From Disk (.E01): Use forensic suites (Autopsy) or specialized tools (RegRipper) that parse hive files (NTUSER.DAT, SOFTWARE).

Hunting Malicious Services

Adversary Techniques:

  • Masquerading: Creating a service with a name that looks legitimate (e.g., MicrosoftUpdateSvc).
  • Binary Path Hijacking: Modifying the ImagePath of a rarely used service to point to malware.
  • New Service Creation: Installing malware directly as a new service, hoping it goes unnoticed.

THE FORENSIC METHOD

  • From Memory: Volatility's windows.svcscan.SvcScan plugin is powerful. It shows all services, state (running/stopped), binary path, and PID. Correlate this with process lists.
  • From Disk: Configurations are stored in the SYSTEM registry hive under SYSTEM\CurrentControlSet\Services. Parse this to check ImagePath and DisplayName.

Hunting in Scheduled Tasks

ADVERSARY USE CASE

Ideal for periodic execution, such as:

  • Checking for a C2 connection every hour.
  • Re-infecting a system if the primary payload is deleted.
  • Running automated cleanup scripts.

THE FORENSIC METHOD

  • On Disk: Tasks are stored as XML files in C:\Windows\System32\Tasks. Examine these to see the , , and . Adversaries often forget to change default authors.
  • In Memory: Less direct, but you can hunt for the result of a scheduled task, like the schtasks.exe process or the spawned malicious process using windows.pslist.PsList.
