Ghidra: The Disassembler & Decompiler

  • What it is: A free, full-featured Software Reverse Engineering (SRE) framework developed and released by the NSA.
  • Core Function (Disassembler): Takes a compiled executable (binary) and translates the machine code back into human-readable assembly language (e.g., x86, ARM).
  • Magic Feature (Decompiler): Attempts to translate the assembly code into a high-level C-like pseudocode. This is our primary tool for understanding program logic quickly.
  • Our Use Case: We use it to perform Static Analysis—analyzing a program's structure, strings, and logic without having to run it.

Volatility 3: The Memory Forensics Framework

Why Memory is King: RAM contains a treasure trove of evidence that may never touch the disk:

  • Running processes (including hidden or injected ones)
  • Active and recent network connections
  • Plaintext passwords, encryption keys, and sensitive data
  • Reflectively loaded DLLs and fileless malware

How it Works: Python framework using a powerful plugin system to parse specific data structures in memory.

Our Use Case: Find evidence of adversary actions as they were happening.

Plaso: The Super Timeline Creator

  • Full Name: "Plaso Langar Að Safna Öllu" (Icelandic for "Plaso wants to collect everything").
  • Core Function: It parses everything—Registry hives, browser history, event logs, file system metadata (MFT)—and aggregates all events into a single chronological file.
  • The "Super Timeline": Building a complete, undeniable narrative of an intrusion from start to finish.

The Integrated Workflow

  1. Start: The Hunt
    Analyze memory with Volatility or disk with Plaso to find suspicious events.
  2. Next: Extract
    Carve the suspicious executable from the memory dump or copy it from the disk image.
  3. Then: Analyze
    Load the executable into Ghidra to reverse engineer logic and find unique indicators.
  4. Finally: Pivot
    Use high-fidelity indicators from Ghidra to launch a new, enterprise-wide hunt.
