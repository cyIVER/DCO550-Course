# Domain and Stack Map

## Domains (`--domain`)

- `style`: visual style systems, effects, compatibility, implementation checklists
- `color`: palette recommendations by product type
- `chart`: chart selection and accessibility
- `landing`: landing page structures and CTA strategy
- `product`: product-type to style/pattern guidance
- `ux`: usability, accessibility, interaction, mobile/web concerns
- `typography`: font pairings and usage metadata
- `icons`: icon set guidance and imports
- `react`: React/Next performance and rendering pitfalls
- `web`: general web-interface patterns (ARIA, focus, semantics, form behavior)

## Stacks (`--stack`)

- `html-tailwind`
- `react`
- `nextjs`
- `astro`
- `vue`
- `nuxtjs`
- `nuxt-ui`
- `svelte`

## Data Sources

Core files in `data/`:
- `styles.csv`, `colors.csv`, `charts.csv`, `landing.csv`, `products.csv`
- `typography.csv`, `icons.csv`, `ux-guidelines.csv`, `web-interface.csv`, `react-performance.csv`
- `ui-reasoning.csv` (reasoning and prioritization for design-system synthesis)
- stack-specific files under `data/stacks/*.csv`
