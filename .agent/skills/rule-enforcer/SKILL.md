# Rule Enforcer

## 1. Definitive Goal Statement
Your objective is to guarantee that every new configuration, code snippet, or `SKILL.md` file generated within this workspace complies flawlessly with the global structural constraints defined in `.agent/rules/`.

## 2. Trigger Conditions and Operational Context
You must invoke this logical sequence when:
- A new skill is generated by the `skill-creator`.
- A user explicitly asks to verify or audit a file against the project rules.
- During the automated CI/CD checks of a workflow scaffold.

## 3. Resource Discovery and Grounding
<available_resources>
- You MUST fetch and read all `.md` files located in the `.agent/rules/` directory (e.g. `enforce-progressive-disclosure.md`) before conducting an audit.
</available_resources>

## 4. Deterministic Execution Logic (Checklist)
You must process the following implementation steps sequentially. Do not proceed to a subsequent step until the current step is fully executed, verified, and its state is recorded.

* [ ] **Step 1: Rule Ingestion.** List the directory contents of `.agent/rules/` and read all rule files present to establish the current enforcement constraints.
* [ ] **Step 2: Target Identification.** Identify the specific file(s) or code snippets that need to be audited for compliance.
* [ ] **Step 3: Auditing.** Systematically verify the target files against EVERY ingested rule. If a file violates a structural constraint (e.g. putting raw reference data in a `SKILL.md` instead of `references/`), flag the exact line and issue.
* [ ] **Step 4: Remediation Proposal.** Provide a strict diff patch to bring the file back into compliance. If the violation is severe but cannot be automatically patched, output a blocking error.
* [ ] **Step 5: File Update.** Use the `replace_file_content` tool to apply your proposed remediation directly to the offending file.

## 5. Strict Constraints (Negative Guardrails)
- **DO NOT** invent new ad-hoc rules during an audit. Only enforce what is documented in the `.agent/rules/` directory.
- **DO NOT** blindly approve files that technically function but violate formatting standards. Standardized syntax is critical to deterministic agent behavior.
- **DO NOT** rewrite the target file from scratch. Apply surgically precise file edits via `replace_file_content` or `multi_replace_file_content`.

## 6. End Task Directive
Once the target files have been audited and patched into compliance, output the exact phrase: "Rule Enforcement Audit Complete. All files compliant. Awaiting next directive."
<END_TASK>
